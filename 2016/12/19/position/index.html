position

CSS 的position属性设置元素的定位方式,为将要定位的元素定义定位规则。默认值为：static

  • 定位元素 是计算后位置属性为relative，absolute，fixed 或 sticky的元素。
  • 相对定位元素 是计算后位置属性为relative的元素
  • 绝对定位元素 是计算后位置属性为absolute 或 fixed的元素
  • 粘贴定位元素 是计算后位置属性为sticky的元素。

top，right，bottom和left属性指定了被定位元素的位置

syntax

  • static
    该关键字指定元素使用正常的布局行为，即元素在文档流中当前的布局位置，此时top, right, bottom, left 和 z-index属性无效
    (默认值，没有定位，元素出现在正常流中)

  • relative
    该关键字下，元素先放置在为添加定位时的位置，再在不改变页面布局的前提下调整位置(因此会在此元素为添加定位时所在位置留下空白).
    (生成相对定位的元素，相对于其在普通流中的位置进行定位)

  • absolute
    不为元素预留空间，通过制定元素相对于最近的非static定位祖先元素的偏移，来确定元素位置。绝对定位的元素可以设置外边距(margins),且不会与其他边距合并。
    (生成绝对定位的元素，相对于最近一级的，定位不是static的元素来进行定位)

  • fixed
    不为元素预留空间，而是通过制定元素对于屏幕视口(viewport) 的位置来制定元素位置。元素的位置再屏幕滚动时不会改变。打印时，元素会出现杂每页的固定位置。fixed属性会创建新的栈上下文。当元素祖先的transform属性非none时，容器由视口改为祖先。
    ((老IE不支持)生成绝对对位元素，通常相对于浏览器窗口或frame进行定位。)

  • sticky
    元素先按照普通流定位，然后相对于该元素在流中的flow root 和 containing block(最近的块级祖先元素)定位。在所有情况下(即便被定位元素为table时)，该元素定位均不对后续元素造成影响。当元素B被粘贴定位时，后续元素的位置仍按照B未定位时的位置来确定。position: sticky 对table元素的效果与position: relative相同。
    (生成粘贴定位元素，容器的位置根据正常文档流计算得出。)

对于相对定位元素，top 和 bottom属性指定它相对于正常位置的垂直偏移，left和right属性指定水平偏移。
对于绝对定位元素，top，right，bottom和left属性指定元素相对于其包含块的偏移，即此时位置为与包含块的相对位置。元素的边距margin定位在这些偏移之中。
在大多数时候，绝对定位元素的height 和 width属性的值为auto，他们会自动计算以合适元素的内容。但是非替换(non-replaced)绝对定位元素可以占据top和bottom的值(除auto外)所有共同制定的可用空间，而不必设置height。

绝对定位元素的定位值发生冲突时的解决方法：

  • 如果同时制定top和bottom，优先采用top
  • 如果同时制定left和right，若derection为ltr，则优先采用left，若derection为rtl，则优先采用right
