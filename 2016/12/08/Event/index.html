EventTarget.addEventListener

Evnet 属性

event.currentTarget

当事件遍历DOM时，识别事件的当前目标对象(Identifies the current target for the evvent,as the event traverses the DOM).该属性总是指向被绑定事件句柄(event handler)的元素。而与之对比的event.target，则是指向触发该事件的元素。

event.currentTarget is interesting to use when attaching the same event handler to several elements

event.target

指向触发事件的对象。当事件处理程序在冒泡阶段或者捕获阶段调用的时候，该目标对象与event.currentTarget不同。(只有当绑定的事件处理程序与触发该事件处理程序都为同一个对象的时候，两者相同。)

在IE6-8中，事件模型与标准不同。使用非标准的element.attachEvent()方法绑定事件监听器。在该模型中，事件对象有一个srcElement属性，等价于target属性。

event.bubbles

返回一个布尔值，表明当前事件是否会向DOM树上层元素冒泡.

syntax

一些特定的事件类型会冒泡。这时，该事件对象的bubbles属性为true。你可以检查该属性的值来判断一个事件对象是否冒泡

demo:

event.cancelable

返回一个布尔值，表明该事件是否可以被取消
syntax:

一个事件的默认动作能否被取消，是在该事件初始化时规定的。
要取消又给事件的默认动作，可以调用该事件的preventDefault()方法

event.defaultPrevented

返回一个布尔值，表明当前事件的默认动作是否被取消，也就是是否执行了event.preventDefault()方法

你应该使用该属性来代替以前的非标准的已经被废弃的getPreventDefault()方法

syntax:

event.isTrusted

返回一个布尔值，表明当前事件是否是由用户行为触发(比如说真实的鼠标点击触发一个click事件)，还是由一个脚本生成的(使用事件构造方法，比如event.initEvent)

syntax

event.type

返回一个字符串，表示该事件对象的事件类型。

传给EventTarget.addEventListener()和EventTarget.removeEventListener()方法的event参数的值是忽略大小写的

event.timeStamp

返回事件发生时的时间戳
